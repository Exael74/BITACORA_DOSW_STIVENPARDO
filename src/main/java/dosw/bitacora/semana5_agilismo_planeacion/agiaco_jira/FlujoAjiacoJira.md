# Preparación de un Ajiaco Santafereño (Flujo en Jira)

**Contexto:** Ejercicio práctico de Agilismo y Planeación (Semana 5).
El objetivo de este ejercicio fue utilizar **Jira Software** para simular la planeación y ejecución de un proyecto mediante historias de usuario. El proyecto elegido fue la preparación de un plato típico colombiano: **El Ajiaco**.

---

## 1. Creación del Backlog
El primer paso consistió en desglosar el gran épico "Preparar Ajiaco" en historias de usuario más pequeñas, manejables y con valor. Estas historias se añadieron al **Backlog** del proyecto en Jira.

![Backlog del Proyecto](./BackLog%20ajiaco.png)
*Vista general del listado de tareas e historias de usuario preparadas para el Ajiaco.*

---

## 2. Detalle de Historias de Usuario (HU)

Cada historia de usuario fue documentada con su respectiva _Descripción_ (Como <rol>, quiero <acción>, para <beneficio>) y _Criterios de Aceptación_.

### HU 1: Compra y alistamiento de Ingredientes
![Detalle HU 1](./HU%201.png)
*La primera historia abarca conseguir las guascas, los 3 tipos de papa, el pollo, el maíz, alcaparras y crema de leche.*

### HU 2: Cocción Base (El Caldo)
![Detalle HU 2](./HU%202.png)
*Preparar el pollo con la base de papa sabanera y pastusa para darle el espesor inicial al caldo.*

### HU 3: Adición de Papa Criolla y Guascas
![Detalle HU 3](./HU%203.png)
*El paso clave para darle el color amarillo y el sabor tradicional al ajiaco mediante la hierba característica.*

### HU 4: Servido y Acompañamientos
![Detalle HU 4](./HU%204.png)
*Desmechar el pollo, servir con arroz, aguacate, crema de leche y alcaparras de manera opcional para el comensal.*

---

## 3. Asignación y Ejecución del Sprint

Una vez refinadas las historias en el backlog, se movieron a un **Sprint Activo**, asignando responsables, puntos de historia (Story Points para estimar complejidad) y fechas de inicio/fin.

![Sprint Asignado](./Sprint%20assigned.png)
*Tablero Kanban activo (To Do, In Progress, Done) mostrando las tareas asignadas para este Sprint.*

---

## 4. Finalización del Sprint

A medida que "el ajiaco se iba cocinando", las tarjetas se movieron por el tablero hasta llegar a la columna *Done*. Una vez completadas todas las iteraciones definidas para el objetivo, se dio por terminado el Sprint.

![Sprint Finalizado](./Sprint%20Finished.png)
*Reporte de Jira evidenciando el cierre exitoso del Sprint y la completitud de las tareas de nuestro Ajiaco.*
