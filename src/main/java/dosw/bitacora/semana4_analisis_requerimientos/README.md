# Semana 4: Análisis de Requerimientos

## Objetivo
Desarrollar habilidades en análisis de requerimientos de software, aplicación de técnicas de levantamiento de requerimientos y documentación formal de especificaciones.

## Contenido

### Ejercicios realizados en clase

#### Diagrama de Contexto de UBER
- **diagrama_contexto_uber/**: Análisis del ecosistema completo de la aplicación UBER
- Identificación de actores externos y sistemas
- Definición de fronteras del sistema

#### Análisis de Requerimientos - Formulario de Inscripción a la Carrera
- **formulario_inscripcion_escuela/**: Análisis completo para sistema de inscripciones

**Documentación requerida detallada:**
- **Nombre**: Identificación del requerimiento
- **Descripción**: Explicación detallada del requerimiento
- **Cómo se ejecutará**: Proceso de implementación
- **Actor Principal**: Usuario primario del sistema
- **Precondiciones**: Estados necesarios antes de la ejecución
- **Datos de entrada y de salida**: Especificación de inputs/outputs
- **Flujo Básico**: Secuencia normal de eventos
- **Flujo Alterno (Error)**: Manejo de excepciones y errores
- **Diagrama de Casos de Uso**: Representación visual
- **Reglas de negocio**: Políticas y restricciones del dominio

#### Manual de Identidad del Estudiante
- **manual_identidad_estudiante/**: Documentación de identidad corporativa estudiantil
- Definición de elementos gráficos y comunicacionales

## Skills desarrolladas
- Análisis de requerimientos de software
