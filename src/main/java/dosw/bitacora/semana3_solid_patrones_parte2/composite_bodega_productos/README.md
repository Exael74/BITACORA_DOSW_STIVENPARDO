# Composite Pattern

## Descripción
El patrón Composite es un patrón de diseño estructural que permite componer objetos en estructuras de árbol para representar jerarquías de parte-todo. Este patrón permite a los clientes tratar objetos individuales y composiciones de objetos de manera uniforme a través de una misma interfaz.

## ¿Cuándo usarlo? (Casos de uso)
- Cuando tienes que implementar una estructura de árbol para representar jerarquías parte-todo (como sistemas de archivos, menús de interfaz gráfica, o estructuras organizativas).
- Cuando quieres que el código cliente trate de la misma forma objetos simples e individuales como a los contenedores complejos que los agrupan.
