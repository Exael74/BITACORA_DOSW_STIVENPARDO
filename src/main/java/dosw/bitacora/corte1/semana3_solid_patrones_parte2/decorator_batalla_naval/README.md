# Decorator Pattern

## Descripción
El patrón Decorator es un patrón de diseño estructural que permite añadir dinámicamente nuevos comportamientos a objetos sin alterar las clases subyacentes. Lo hace envolviendo el objeto dentro de un objeto decorador ("wrapper") que contiene la funcionalidad adicional.

## ¿Cuándo usarlo? (Casos de uso)
- Cuando necesitas añadir responsabilidades o comportamientos adicionales a objetos individuales de forma dinámica y transparente durante el tiempo de ejecución.
- Cuando la herencia no es una opción viable o resulta en una explosión exponencial de subclases (para combinar varios comportamientos).
- Cuando se requiere retirar temporalmente las responsabilidades del objeto.
